# как растёт этот репозиторий

s26-lab — публичная витрина лабы, которая наполняется по мере её развития. репо не пишется разом: после каждой открытой сессии сюда добавляется то, что можно показать публично. этот файл описывает, как именно.

## принцип: публичное отделено от приватного

рабочий контекст лабы (переговоры, договорённости, личные данные участников и спикеров) живёт в приватном рабочем пространстве. в этот репозиторий попадает **только публичная проекция**.

**граница «уже публичного» — сайт лабы.** если что-то показано на [s26.aimindset.org](https://s26.aimindset.org/), это можно отразить и здесь. всё, чего на сайте нет, сюда не идёт.

| попадает в репо | остаётся приватным |
|-----------------|---------------------|
| программа: шесть элементов, недели, форматы, исходы | цены, пакеты, условия оплаты |
| принципы и методология | контакты, личные хендлы |
| обезличенные скиллы (без ключей и личных путей) | статусы переговоров, внутренние договорённости |
| проводники, уже заявленные на сайте (имя + роль) | неподтверждённый состав, не показанный публично |
| артефакты сессий (диаграммы, обезличенные плагины) | сырые транскрипты, личные данные |

## куда что класть

| папка | содержимое |
|-------|------------|
| `program/` | программа, шесть элементов, недели, ритм, карта подходов, проводники |
| `principles/` | принципы работы лабы |
| `streams/` | открытые эфиры (публичная сборка) |
| `skills/` | обезличенные скиллы-методологии |
| `glossary/` | термины лабы |
| `artifacts/` | артефакты сессий |
| `assets/` | shaper-диаграммы + `shaper.css` |

## стиль

все артефакты — в эстетике **shaper** (см. [`STYLE.md`](STYLE.md)): чёрно-белый технический язык, моноширинный шрифт, строчные буквы, рамки 1px, инверсия как активное состояние. новый артефакт делается по шаблону из `assets/` — так весь репозиторий читается как единая система.

## как контрибьютить

репозиторий публичный:
- **форк → правки → pull request** — работает сразу, без ожидания доступа;
- участники и проводники лабы с прямым доступом могут пушить в ветку и открывать PR.

перед каждым добавлением — проверка, что не утекли личные данные, контакты, ключи или неопубликованный состав. в репо идёт только обезличенное и уже публичное.
